async Task dataPARC.Store.SDK.PublishingClient.StopPublishingAsync ( bool  force = false,
CancellationToken  token = default 
)

Stop publishing datapoints for all active requests.

Parameters
forceIndicates if publishing should stop abruptly. If true, and results were being cached on the client due to it being backed up, then the callback will not be run for the cached results and all publishing activity will end immediately. If false, and results for these subscriptions were being cached on the client, then all cached data will be processed before publishing stops. In this case, no new data will be published from the server, but any data already received will be processed by the callback.
tokenA cancellation token that can be used to cancel the operation. If unspecified, no token is attached to the request.
Returns
A task the represents the asynchronous operation, which wraps a bool. The value indicates if publishing was successfully stopped for all active requests specified.

Initializes a new instance of the ReadClient class with an array of hostnames and port numbers.

The first server in the array of servers is set as the primary server.

Parameters
serversThe array of hostname and port number pairs.
certificateValidationThis provides methods for validating the server's certificate.
authProviderIf authentication is enabled, this is the OAuth provider for security tokens.
enableInternalLogMessagesTurn on internal communication logging used for troubleshooting.
deadlineSecondsThe default timeout for requests, in seconds. This is attached to all requests that specify a timeout, even retries to failover servers. This means a read request on the client may go longer than this value without timing out since the time restarts with each retry.
avgFailoverMinutesThe average number of minutes to wait once failover occurs before trying to revert to the primary server.

Stream aggregated datapoints for a given time period.

This method is equivalent ReadProcessedAsync in the data it can return, but starts returning data faster since the server sends data into the stream as soon as it starts reading it. It also provides more flexibility around client memory constraints since the stream can be consumed without needing to bring all datapoints being read into memory at once on the client.

parametersParameters for streaming aggregated datapoints.
cancellationTokenA cancellation token that can be used to cancel the operation. If unspecified, no token is attached to the request.
Returns
A task that represents the asynchronous read operation, which wraps a StreamProcessedResult. Instead of containing all the datapoints requested, the result object has methods for streaming the datapoints from the server.

Stream raw datapoints for multiple tags.

This method is equivalent ReadRawBulkAsync in the data it can return, but starts returning data faster since the server sends data into the stream as soon as it starts reading it. It also provides more flexibility around client memory constraints since the stream can be consumed without needing to bring all datapoints being read into memory at once on the client.
